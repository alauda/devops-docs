# 介绍

## Alauda DevOps 介绍

Alauda DevOps 是一款基于 Kubernetes 构建的云原生 DevOps 平台，旨在快速构建和交付云原生应用及工作负载。Alauda DevOps 为从代码到生产提供了完整的 DevOps 工具链，包括源代码管理、工件和依赖管理、持续集成、持续交付以及源代码质量控制。

## 优势

Alauda DevOps 的核心优势如下：

- 提高运营效率

  通过自动化和简化 DevOps 工作流程，该平台减少了手动操作，最大限度降低错误，加速软件交付。团队可以专注于创新，同时保持高质量标准。

- 安全优先的方法

  该平台以安全为基础，确保敏感数据、构建和部署的端到端保护。它与企业安全实践无缝集成，包括加密、访问控制和合规检查。

- 可扩展性

  该平台在灵活性和自定义方面进行了优化，使得组织能够整合多种工具、服务和平台。借助 Tekton 的模块化和可重用组件以及连接器提供的安全性和可扩展性，团队可以轻松扩展管道，以满足不断变化的需求。

- 统一工作流

  团队可以集中管理他们的 DevOps 过程，形成构建、集成和部署的单一真实来源。这减少了复杂性并促进了开发、运维和安全团队之间的更好协作。

- 更快的上市时间

  通过自动化 CI/CD 过程和利用预构建的连接器，组织能够加速高质量应用的交付，缩短新功能和服务的上市时间。

- 先进的 DevOps 实践

  团队可以采用并完善现代化的 DevOps 策略，如事件驱动的管道、管道即代码及先进监控，以优化他们的开发和交付过程。

## 场景

- 平台搭建

  平台团队可以使用 Alauda DevOps 来构建和管理自己的 DevOps 平台，使业务团队专注于应用开发和交付，同时保持高质量标准和安全性。

- 精简的 CI/CD 管道

  团队可以快速开始使用黄金路径管道交付应用，并定制管道以满足特定需求，获取实时反馈和见解。

- 集中的自动化工作流

  通过创建和共享可重用的工作流和任务，团队可以使用自动化和可重复的工作流提高效率，减少手动过程引起的错误。

## 限制

- 开箱即用的工作流和任务主要聚焦于云原生应用及工作负载，可能需要额外开发以支持其他类型的应用。

- 连接器当前支持 `git` 和 `oci` 协议，并对身份验证方法的支持有限。
