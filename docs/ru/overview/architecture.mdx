# Архитектура

![](/architecture.drawio.png)

Alauda DevOps состоит из нескольких надстроек, чтобы создать полную платформу DevOps, включая:

- <ExternalSiteLink name="alauda-devops-pipelines" children="Потоки Alauda DevOps" />: Облачные нативные потоки и управление задачами на основе Tekton
- <ExternalSiteLink name="alauda-devops-connectors" children="Коннекторы Alauda DevOps" />: Безопасный и эффективный способ подключения к внешним услугам и инструментам DevOps
- <ExternalSiteLink name="alauda-build-of-gitlab" children="Сборка Alauda Gitlab" />: Платформа DevOps с открытым исходным кодом на основе веб-технологий, предоставляющая управление исходным кодом, непрерывную интеграцию и возможности отслеживания проблем для команд разработки программного обеспечения.
- <ExternalSiteLink name="alauda-build-of-harbor" children="Сборка Alauda Harbor" />: Облачный нативный реестр, который безопасно хранит, подписывает и сканирует образы контейнеров с управлением доступом на основе ролей, сканированием уязвимостей и возможностями доверия к содержимому.
- <ExternalSiteLink name="alauda-build-of-sonarqube" children="Сборка Alauda SonarQube" />: Продвинутая платформа для обеспечения качества и безопасности кода, которая интегрирует возможности на основе ИИ для обнаружения и проверки кода, созданного ИИ, предлагает повышенную безопасность с комплексным обнаружением уязвимостей и обеспечивает более высокую продуктивность разработчиков за счет сокращения времени анализа и бесшовной интеграции с IDE.
- <ExternalSiteLink name="alauda-build-of-nexus" children="Сборка Alauda Nexus" />: Комплексный менеджер бинарных репозиториев, который служит единым источником правды для всех внутренних и сторонних бинарных файлов, компонентов и пакетов.
